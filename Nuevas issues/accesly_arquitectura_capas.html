Capa 01 · CanalesMeta · Telegram · IG
Canales de mensajería
Reciben mensajes, muestran el cobro interactivo y disparan el WebView. Una identidad por número de teléfono sirve en todas las redes.
WhatsApp Cloud API / BSP Telegram Bot API Instagram Messaging Webhooks entrantes Botón URL · Flows
Capa 02 · WebViewAccesly
Pantalla de pago (WebView embebido)
Front web propio servido dentro del chat. La única capa donde Accesly controla el diseño al 100%: es donde vive la experiencia del demo.
Front propio (HTML) Passkey / biometría Confirmación de pago Cripto invisible
Capa 03 · NúcleoAccesly
Orquestación (Accesly Core)
El cerebro. Crea la intención de cobro, coordina ramp y liquidación, aplica el partner fee y notifica al canal. Nunca custodia el bruto.
API Gateway Motor de cobros Máquina de estados Idempotencia Partner fee Ledger espejo
Capa 04 · IdentidadAccesly
Identidad y wallet (Account Abstraction)
Wallet no custodial creada con passkey y teléfono, sin frase semilla. El quórum de firma siempre exige al usuario. Accesly patrocina el gas.
WebAuthn / Passkey MPC 2-de-3 KMS / HSM Recuperación por OTP Gas patrocinado
Capa 05 · AgentesAccesly
Agentes de IA y herramientas
Atienden, cobran y ejecutan acciones. El consumo se mide contra un saldo de IA. Las herramientas se conectan vía MCP + OAuth.
Orquestador de agente Metering / saldo IA Tool use (function calling) MCP OAuth (Composio) Calendar · Gmail · Sheets · Drive
De aquí hacia abajo, invisible para el usuario
Capa 06 · RampEtherfuse · regulado
On / off-ramp regulado
Convierte fiat y USDC, es el sujeto obligado que hace KYC/KYB y mueve el dinero por SPEI. Mantiene a Accesly fuera del perímetro regulatorio.
Onramp (CLABE SPEI) Offramp Swap KYC / KYB Claimable balance
Capa 07 · LiquidaciónStellar · Circle
Liquidación on-chain
Asienta el valor en segundos, con costo casi nulo y de forma atómica. El split del partner fee viaja en la misma transacción.
USDC (Circle) Trustlines Sponsored reserves Fee-bump Liquidación atómica
No custodia
El dinero nunca vive en una cuenta de Accesly. Fluye del cliente al negocio.
Blockchain invisible
Ni wallets, ni gas, ni tokens. El cliente paga con biometría.
Mínima fricción
Passkey y teléfono. Pago en uno o dos toques. Cero frase semilla.
Flujos atómicos
Cada cobro se completa o se revierte como una unidad idempotente.
Lectura: capas 02 a 05 son de Accesly · capa 01 es el canal (Meta) · capas 06 y 07 son terceros (Etherfuse, Stellar). La frontera marca dónde empieza la capa cripto, oculta al usuario final.
